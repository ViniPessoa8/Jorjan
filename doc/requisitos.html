Requisitos Não-Funcionais

Os requisitos não-funcionais descrevem necessidades em relação ao contexto onde o sistema executará.

# Descrição Métrica
RNF001 Desempenho As ações do serviço devem ocorrer em menos de 3 segundos.
RNF002 Disponibilidade O serviço deve estar disponível no mínimo de 6:00h até 24:00h.
RNF003 Documentação O sistema deve estar bem documentado.
RNF004 Segurança Os dados dos usuarios devem estar seguros de vazamentos.
RNF005 Portabilidade O sistema deve executar igualmente, tanto para web quanto para Android.
RNF006 Usabilidade O usuário deve conseguir realizar um pedido em menos de 5 minutos.
RNF007 Manutenção O código deve ser padronizado, comentado e documentado.
RNF008 Compatibilidade

O sistema deve ser compátível com versões Android acima da 6.0.

O sistema deve ser compatível com os navegadores: Mozilla Firefox, Google Chrome e Safari.
RNF009 Suporte O suporte deve ser feito através de troca de e-mails.
RNF010 Tráfego O sistema deverá suportar 2000 usuários simultâneos.
RNF011 Tamanho da tela O sistema deverá suportar telas a partir de 1280x720.
RNF012 Tamanho da senha As senha devem ter de 6 a 16 caracteres.
RNF013 Tamanho do nickname O nickname deve ter de 6 a 16 caracteres.

Regras de Negócios

As regras de negócio definem como o sistema se comporta perante limitações do contexto onde está inserido.

# Descrição
RN001 Para efetuar o cadastro de um usuário, o sistema deve validar os dados.
RN002 Os vendedores terão prioridade na listagem de acordo com a sua avalição geral.
RN003 O usuário não poderá ter mais de um perfil vinculado ao seu e-mail.
RN004 O cancelamento de pedidos deve ser feito antes da confirmação.
